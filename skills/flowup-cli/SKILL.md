# Flowup CLI

Use this skill when the task involves the local `@wry-smile/flowup` CLI or packages generated by it.

## When To Use

- The user wants to generate a Node-RED node or plugin scaffold with `flowup gen`
- The user wants to build a Flowup package with `flowup build`
- The user wants to assemble multiple Flowup-built packages into one distributable package with `flowup assemble`
- The user wants to edit `flowup.config.ts`, template generation logic, or Flowup client SDK helpers

## Command Model

- `flowup gen`
  Generates Node-RED node or plugin templates
- `flowup build`
  Runs Vite build twice with `runtime` and `editor` modes
- `flowup assemble`
  Scans Flowup-built packages, optionally builds them, then assembles their `dist/` outputs into one package

## Configuration Rules

- Use `flowup.config.ts` as the shared configuration entry
- Package-level config commonly defines:
  - `scope`
  - `type`
  - `runtime`
  - `client`
  - `package`
- Assemble-level config lives under `assemble` in the same `flowup.config.ts`

Example:

```ts
import { defineConfig } from '@wry-smile/flowup'

export default defineConfig({
  scope: 'demo-node',
  type: 'nodes',
  assemble: {
    output: 'dist/node-red-assemble',
    name: 'node-red-demo-pack',
  },
})
```

## Template Expectations

- Generated packages should keep Node-RED-oriented folders such as `runtime/`, `client/`, `types/`, `locales/`, `icons/`, and `resources/`
- Framework choices currently include `vanilla`, `svelte`, and `vue`
- `tailwind` is only relevant for framework-based client templates, not vanilla templates

## Working Conventions

- Prefer editing template behavior in `cli/src/templates/`
- Prefer editing command behavior in `cli/src/commands/`
- Prefer `flowup.config.ts` over introducing extra config entry files
- When removing redundant ESM import suffixes in source templates, keep `package.json` output file references intact unless the runtime consumer also supports extensionless paths

## Verification

- For CLI source changes, run `./node_modules/.bin/tsc --noEmit` in `cli/`
- Rebuild with `./node_modules/.bin/tsdown` in `cli/` when command output or exported API changes
- Use `node cli/dist/bin/flowup.js --help` or subcommand help to verify CLI wiring
