# Flowup

A pnpm monorepo for building the `@wry-smile/flowup` CLI and maintaining example Node-RED nodes and plugins generated by it.

## Repository Layout

```text
.
├── cli/                 # Flowup CLI source, docs, and published consumer skills
├── packages/
│   ├── nodes/           # Example Node-RED node packages
│   └── plugins/         # Example Node-RED plugin packages
├── skills/              # Repository-level skills for working on Flowup itself
├── pnpm-workspace.yaml
└── package.json
```

## What Flowup Provides

The CLI currently focuses on three workflows:

- `flowup gen`
  Scaffold a Node-RED node or plugin package
- `flowup build`
  Build a Flowup package with Vite in `runtime` and `editor` modes
- `flowup assemble`
  Assemble multiple Flowup-built packages into one distributable Node-RED package

Detailed CLI usage lives in [cli/README.md](./cli/README.md).

## Workspace Packages

This repository includes example generated packages under:

- `packages/nodes/simple-node`
- `packages/plugins/simple-plugin`

They reflect the current Flowup template structure:

- `flowup.config.ts`
- `runtime/`
- `client/`
- `constant/`
- `types/`
- `locales/`
- `icons/`
- `resources/`

Plugin templates are intentionally lightweight and do not require a generated `client/editor.html`.

## Common Commands

Install workspace dependencies:

```bash
pnpm install
```

Build the CLI:

```bash
pnpm build
```

Generate a new node example:

```bash
pnpm gen:node
```

Generate a new plugin example:

```bash
pnpm gen:plugin
```

Build an example package:

```bash
cd packages/nodes/simple-node
flowup build
```

Assemble packages from the repository root:

```bash
cd /path/to/flowup
flowup assemble
```

## Configuration

Flowup uses `flowup.config.ts` as the shared configuration entry for package builds and assemble behavior.

Example:

```ts
import { defineConfig } from '@wry-smile/flowup'

export default defineConfig({
  scope: 'my-node',
  type: 'nodes',
  assemble: {
    output: 'dist/node-red-assemble',
    name: 'node-red-my-assemble',
  },
})
```

## Notes

- The CLI package is published from `cli/`
- The root package is a private workspace container
- Consumer-facing Flowup skills are shipped from `cli/skills/`
- Repository-level development skills live under `skills/`
